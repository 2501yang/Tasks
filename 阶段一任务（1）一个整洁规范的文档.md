| 1. *既然你选择了 Geek 组，那么你觉得，什么是 Geek 精神？*    |
| Geek说“虚掩着的门是一种挑衅，锁着的门是一种侮辱”这句话真的说到我心上了，新世界的大门就在眼前，而Geek精神就是不怕困难多，相信好奇之心，改变之力，通过努力一定能成功。不是一定要成为大神，而是用代码把自己的想法变成现实，哪怕只是做了一个有一点用处的小工具，也已经用代码，悄悄改变了生活的某个角落。 |
| 2. *你认为 `Github` 是干啥的？里面那些`Pull request`，`Fork`之类的词语又是干啥的？这些你又是通过哪些途径学到的？* |
| 我认为 `Github` 是学习 + 展示平台，可以找到大佬的开源代码学技术，或自己的项目放上去展示。`Fork `是复制别人的仓库到自己的账户下，这样自己就可以在不影响原仓库的情况下对代码进行修改。`Pull Request `是当自己对代码做了修改后，向原仓库作者发起请求，希望对方能将自己的修改并到原仓库中。我在`Github`官方新手指南文档和浏览器查阅学到这些的。 |
| 3. *有学长一开始了解 `Github` 时，光是进网站就花了半天，你遇到过这种情况吗？如果遇到过，你又是如何解决的？如果没遇到过，那你觉得学长的情况是因为什么？* |
| 遇到过，可能是因为服务器在国外，网络延迟高。我请教同学通过下载加速器解决了这个问题。 |
| 4.*最后，说说你的这份文档有什么理由称得上整洁规范？你又遵循了哪些规范？* |
| 我开头用了一级标题，二级标题，醒目地概括出下面的内容。没有用复杂格式，用有序列表，保持缩进一致。我少用空格/换行，删除“无意义符号”而是用加粗斜体代替。我遵循了结构完整，内容清晰有条理，避免冗余等规范。 |
